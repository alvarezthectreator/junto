# Admin Dashboard Feature Map

This file is a simple readout of the features that make sense in the admin dashboard for this project.

## Already Worth Keeping In Admin

These are the strongest admin-facing areas already present in the app:

1. Active Events
2. User Directory
3. Fraud and Verification Review
4. Venue Management
5. Celebrity Management
6. Safety and Reports
7. Premium and Billing Oversight
8. System Health and Release Status
9. Audit Trail and Session Activity

## Suggested Feature Names For The Dashboard

If you want the admin dashboard to feel organized, these are clean section names:

- `Active Events`
- `Registered Users`
- `High-Risk Accounts`
- `Pending Reviews`
- `Venue Control`
- `Celebrity Control`
- `Safety Queue`
- `Payments And Plans`
- `System Status`
- `Activity Log`

## What Each Section Should Show

### Active Events

Show every currently active event with:

- Event title
- Host name
- City and address
- Date and time
- Capacity
- Event type
- Billing tier
- Status

### Registered Users

Show all users in a scrollable table with:

- Display name
- Username or profile ID
- Email
- Phone
- City
- Occupation
- Joined date
- Active or inactive state
- Verification status

### High-Risk Accounts

Show accounts that need manual review:

- Risk score
- Behavior score
- Identity score
- Flag count
- Verification status
- Last updated

### Pending Reviews

Use this for moderation work:

- Reported users
- Reported events
- Reported venues
- Report reason
- Review status
- Action taken

### Venue Control

Show venue management details:

- Venue name
- Category
- City
- Active state
- Reviews
- Photo count
- Edit and delete actions

### Celebrity Control

Show celebrity booking data:

- Celebrity name
- Category
- Base price
- Currency
- Active state
- Booking count
- Edit and delete actions

### Safety Queue

Show the safety workflow:

- Safety reports
- Blocked accounts
- SOS events
- Check-in issues
- Trusted contacts
- Escalation status

### Payments And Plans

Show monetization and plan data:

- Subscription status
- Premium plan usage
- Failed payments
- Receipts
- Billing tier distribution

### System Status

Show operational health:

- API health
- WebSocket health
- Notification delivery
- Crash report count
- Deployment readiness

### Activity Log

Show what happened in the system:

- Login activity
- Admin actions
- Profile changes
- Event creation and deletion
- Moderation actions

## Best Admin Order

If you want the dashboard to feel practical, I’d order it like this:

1. Active Events
2. Registered Users
3. High-Risk Accounts
4. Pending Reviews
5. Safety Queue
6. Venue Control
7. Celebrity Control
8. Payments And Plans

10. Activity Log

## Short Version

The most deserving admin features for this project are:

- Live active events
- Scrollable user management
- Risk review and verification
- Safety moderation
- Venue and celebrity CRUD
- Billing and premium oversight
- System health monitoring

That gives the admin dashboard real operational value instead of just looking decorative.
